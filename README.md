# PythonQrscanner

Утилита для массовой обработки сканов работ с QR-кодами: распределение по студентам, архивация, проверка email и рассылка архивов.

## Что умеет

- Переименовывать файлы по найденным QR/штрихкодам
- Раскладывать сканы по папкам студентов на основе CSV
- Создавать архивы по папкам студентов
- Отправлять архивы студентам по email
- Валидировать email из CSV
- Работать в GUI (PySide6)
- Работать в фоновом watch-режиме (реакция на новые/изменённые/перемещённые файлы)
- Избегать повторной обработки в watch-режиме (state SQLite)

## Быстрый старт

1. Установите зависимости:

   ```bash
   pip install -r requirements.txt
   ```

2. Для macOS установите системные библиотеки:

   - `zbar` (для `pyzbar`)
   - `poppler` (для обработки PDF)

   ```bash
   brew install zbar poppler
   ```

3. Пример запуска сортировки по CSV (action=1):

   ```bash
   python main.py --action 1 --image-folder ./images --csv-path ./students.csv --name-fields Фамилия Имя --code-field код --output-folder ./output --move-mode copy --threads 6
   ```

4. GUI:

   ```bash
   python main.py --gui
   ```

## Аргументы CLI

- `--action` — режим работы:
  - `0`: переименование внутри папки
  - `1`: перенос/копирование по CSV
  - `2`: архивация папок студентов
  - `3`: email-рассылка архивов
  - `4`: валидация email
  - `5`: watch-режим (непрерывная сортировка)
- `--image-folder` — входная папка со сканами
- `--csv-path` — CSV с данными студентов
- `--name-fields` — список колонок, из которых собирается имя студента (например `Фамилия Имя`)
- `--code-field` — шаблон имени колонки с кодами (по умолчанию `код`)
- `--email-field` — имя (или часть имени) колонки с email (по умолчанию `email`)
- `--output-folder` — выходная папка
- `--move-mode` — `copy` или `move`
- `--threads` — число потоков/воркеров
- `--gui` — запуск графического интерфейса

## Полная логика режимов

### Action 0 — Переименование внутри папки

1. Сканируются файлы в `--image-folder`.
2. Из каждого файла извлекается QR/штрихкод.
3. Файл переименовывается с учётом найденного кода.

Используется, когда нужно быстро привести имена файлов к системному формату, без CSV-распределения.

### Action 1 — Перенос/копирование по CSV

1. Сканируются файлы из `--image-folder`, извлекаются коды.
2. Читается CSV:
   - строится имя студента из колонок `--name-fields`;
   - находятся все колонки, содержащие `--code-field`.
3. Формируется сопоставление «код → студент».
4. Для файлов:
   - если код найден и сопоставлен студенту — файл попадает в папку этого студента;
   - если код не найден в CSV — файл уходит в `notfound`;
   - если QR не распознан — файл уходит в `noqrcode`.
5. Операция выполняется в режиме `copy` или `move`.

### Action 2 — Архивация

1. Берутся папки студентов внутри `--output-folder`.
2. Для каждой папки создаётся zip-архив.

Режим удобен как отдельный шаг перед рассылкой.

### Action 3 — Email-рассылка

1. Сначала создаются/обновляются архивы папок студентов (как в action 2).
2. CSV читается повторно для получения email:
   - имя студента собирается из `--name-fields`;
   - колонка email ищется по `--email-field` (сначала точное совпадение заголовка, затем частичное).
3. Для каждого архива:
   - имя архива/папки студента используется как ключ студента;
   - по этому ключу берётся email из CSV;
   - если email пустой/не найден — отправка для этого студента пропускается;
   - если email есть — отправляется письмо с zip-вложением.
4. В конце возвращается статистика `sent/failed`.

#### Как именно программа сопоставляет «папку → ученик → email»

- На этапе сортировки (action 1 или watch) создаются папки с именем студента, собранным из `--name-fields`.
- На этапе архивации создаётся архив с тем же ключом имени студента.
- На этапе рассылки программа ищет email по **этому же имени студента**, собранному теми же `--name-fields` из CSV.

Важно: если формат имени в папке и CSV отличается (лишние пробелы, другой порядок колонок, и т.п.), соответствие нарушится и письмо не уйдёт.

### Action 4 — Валидация email

1. CSV читается аналогично action 3:
   - имя студента по `--name-fields`;
   - email по `--email-field`.
2. Для каждого email:
   - проверяется формат адреса;
   - выполняется SMTP-проверка (`NOOP`) с учёткой из SMTP-конфига.
3. В лог выводятся валидные и невалидные адреса.

### Action 5 — Watch-режим (фоновая непрерывная сортировка)

1. Запускается наблюдение за `--image-folder`.
2. Отслеживаются события появления, изменения и перемещения файлов.
3. Для защиты от «недописанных» файлов применяется debounce и проверка стабильности размера/mtime.
4. Пакет готовых файлов передаётся в ту же логику сортировки, что и в action 1.
5. Для дедупликации используется `output/state/processed.sqlite`:
   - проверка дубликатов по хэшу файла;
   - проверка повторного QR для того же хэша;
   - запись статусов обработки.

Примечание: watch-режим ожидает путь к папке. Если передан путь к файлу, сервис автоматически переключается на родительскую папку и пишет предупреждение в лог.

## SMTP-конфигурация

Рекомендуется `.env` в корне проекта:

```env
SMTP_EMAIL=your@email.com
SMTP_PASSWORD=your_password
SMTP_HOST=smtp.example.com
SMTP_PORT=465
```

`sendconfig.py` остаётся fallback-вариантом для обратной совместимости.
