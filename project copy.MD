# ВВЕДЕНИЕ

Целью проекта являлось создание программы, которая распределяет графические файлы (скан-образы работ, содержащих идентификаторы в форме QR-кодов) по папкам на основании таблицы соответствия между кодами и данными участника (фамилия, имя, адрес электронной почты и т. д.), а также архивирует полученные папки и рассылает архивы участникам по электронной почте.

Актуальность задачи обусловлена проведением тестирующих мероприятий (вступительные и аттестационные работы, олимпиады) с анонимизацией участников и необходимостью последующей дешифровки и персонализации результатов. В настоящий момент эти работы осуществляются вручную. Созданная программа позволяет автоматизировать весь процесс: от распознавания QR-кодов до доставки файлов каждому участнику.

Программы с требуемым функционалом слабо представлены на рынке. Имеющиеся аналоги, например, Barcode Utility, предназначены для фиксированных моделей сканеров и МФУ, т. е. не являются аппаратно-независимыми, и не поддерживают автоматическую рассылку по электронной почте.

# ОПИСАНИЕ ПРОГРАММЫ

Программа реализована на языке Python.

## Технические требования:

- установленный интерпретатор Python версии не ниже 3.12
- библиотеки: Pillow, pyzbar, opencv-python, numpy, PySide6, watchdog
- сканы работ в форматах JPG или PNG
- таблица соответствия кодов и участников в формате CSV

## Режимы работы программы

Программа запускается либо через командную строку (`main.py`), либо через графический интерфейс (`--gui`). Предусмотрено шесть режимов (действий):

| Действие                 | Описание                                                                                                                  |
| ------------------------ | ------------------------------------------------------------------------------------------------------------------------- |
| **0 — Переименование**   | Переименовывает файлы изображений по значению декодированного QR-кода                                                     |
| **1 — Распределение**    | Распознаёт QR-коды и раскладывает файлы по папкам участников; нераспознанные — в `unsorted/`, без QR-кода — в `noqrcode/` |
| **2 — Архивирование**    | Упаковывает каждую папку участника в отдельный ZIP-архив                                                                  |
| **3 — Рассылка**         | Выполняет архивирование, затем отправляет каждый архив участнику на адрес электронной почты из CSV                        |
| **4 — Проверка e-mail**  | Проверяет корректность адресов электронной почты из CSV без отправки                                                      |
| **5 — Режим наблюдения** | Непрерывно отслеживает папку и обрабатывает новые/изменённые файлы по мере их появления                                   |

## Подробное описание этапов работы

### Этап 1 — Ввод данных

Программа запрашивает:

- путь к таблице участников в формате CSV;
- путь к папке с исходными файлами (скан-образами);
- путь к папке для результатов;
- разделитель CSV (по умолчанию определяется автоматически);
- названия столбцов с именем участника, кодом и адресом e-mail;
- режим обработки файлов (копирование или перемещение);
- число рабочих потоков.

### Этап 2 — Обработка таблицы участников

Таблица читается с помощью модуля `csv`. Поддерживаются:

- автоматическое определение разделителя (запятая, точка с запятой, табуляция);
- файлы с маркером BOM (кодировка `utf-8-sig`);
- нечёткое сопоставление названий столбцов (без учёта регистра, подстроки);
- несколько столбцов с кодами на одного участника (например, `код-1`, `код-2`);
- кэширование результата по времени изменения файла.

### Этап 3 — Распознавание QR-кодов

Распознавание выполняется с помощью библиотек **Pillow**, **pyzbar** и **OpenCV**:

1. Изображение декодируется в исходной ориентации.
2. Если QR-код не найден, программа перебирает повороты на ±10–180° в три прогрессивных этапа и останавливается при первом успехе.
3. При небольших партиях (≤ 3 файлов) используется `ThreadPoolExecutor`, при больших — `ProcessPoolExecutor` для параллельной обработки.

### Этап 4 — Распределение файлов

По результатам сопоставления QR-кодов с таблицей:

- файлы копируются или перемещаются в папку `<Фамилия_Имя>/` внутри папки результатов;
- файлы с нераспознанным кодом — в `unsorted/`;
- файлы без QR-кода — в `noqrcode/`.

### Этап 5 — Архивирование (действия 2 и 3)

Каждая папка участника упаковывается в отдельный ZIP-архив. Если архив уже существует и новее всех файлов в папке, повторная упаковка пропускается. Архивирование выполняется параллельно через `ThreadPoolExecutor` с помощью библиотеки `zipfile`.

### Этап 6 — Рассылка по электронной почте (действие 3)

После архивирования программа:

1. Считывает адреса электронной почты из CSV-таблицы.
2. Проверяет их корректность (regex-валидация по RFC).
3. Отправляет каждому участнику письмо с прикреплённым ZIP-архивом через SMTP с шифрованием SSL (с помощью библиотеки `smtplib`).
4. Учётные данные SMTP задаются в конфигурационном файле `sendconfig.py`.

### Этап 7 — Режим наблюдения (действие 5)

Сервис наблюдения (`watchdog`) отслеживает папку в реальном времени:

- обнаруживает новые и изменённые файлы;
- ждёт стабилизации файла (настраиваемая задержка) перед обработкой;
- использует SHA-256-хэши и SQLite-базу состояния для пропуска уже обработанных файлов и дубликатов;
- собирает статистику по каждому пакету: получено, обработано, дубликаты, нераспознанные.

## Графический интерфейс

Программа имеет полноценный GUI на базе **PySide6**:

- все параметры задаются через поля ввода с кнопками выбора файлов/папок;
- каждое действие — на отдельной вкладке с кнопкой запуска, прогресс-баром и статусной строкой;
- все операции выполняются в фоновом потоке без «заморозки» интерфейса;
- прогресс отображается в реальном времени: выполнено/всего, скорость, прошедшее время, оценка оставшегося времени;
- вкладка режима наблюдения содержит расширенные настройки: задержку, интервалы, флаги обработки существующих файлов.

# Приложение 2

## Требования к структуре таблицы (csv файл):

1. Таблица должна содержать заголовок (первую строку), в котором перечислены названия полей.
2. Столбцы, содержащие слово "код", не должны повторяться (например, `"код-1","код-2"` допустимы, а `"код-1","код-1"` — нет).
3. Для рассылки по e-mail таблица должна содержать столбец с адресом электронной почты.

Пример строки:

```
Фамилия,Имя,Отчество,email,код-123,код-463
```

Важно:

- разделитель устанавливается в GUI или как аргумент при запуске программы; по умолчанию определяется автоматически
- порядок полей не важен

Таблица может содержать дополнительные поля, но они не обрабатываются программой, если не заданы в настройках.
